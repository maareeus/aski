Control Plane di Aski

Il pannello del proprietario della piattaforma: configura Stripe, definisci i piani e i server, gestisci i tenant e il ciclo di vita delle istanze.

Versione 1.0 · Giugno 2026

Introduzione

Il Control Plane è l’applicazione di amministrazione di Aski. Da qui controlli tre cose:

  • Billing: chiavi Stripe e listini (piani).
  • Infrastruttura: server/regioni dove vengono ospitate le istanze.
  • Clienti: tenant, progetti e relativo provisioning.

La fatturazione pilota l’infrastruttura: quando un abbonamento è attivo il progetto viene avviato, quando è insoluto/disdetto viene fermato (senza perdere i dati).

Ordine consigliatoStripe → Piani → Server → Tenant → Progetto → Checkout/Provisiona.

Avviare l’ambiente

Lo script avvia Postgres, applica le migration e lancia Control Plane e istanza Ticketing.

# Avvia tutto
.\scripts\start-env.ps1

# Opzioni
.\scripts\start-env.ps1 -Fresh       # ricrea il DB da zero
.\scripts\start-env.ps1 -NoBrowser   # non aprire il browser

# Ferma tutto
.\scripts\stop-env.ps1               # -RemoveDb per cancellare anche il DB

Modalità di provisioning

Impostazione Provisioning:Mode in src/Aski.ControlPlane/appsettings.json. Decide cosa accade quando un progetto viene provisionato.

ModalitàCosa serveEffetto
Logging (default)nienteSimulato: il progetto passa a Running senza container reali. Ideale per provare i flussi.
DockerVPS con Docker + TraefikReale: crea container Postgres + app e configura il routing Traefik.
ConsiglioInizia in Logging per imparare i flussi; passa a Docker quando hai una VPS pronta.

Configurare Stripe

Menu Stripe (/StripeAdmin). Servono le chiavi del tuo account Stripe.

1. Recupera le chiavi (modalità Test)

Su dashboard.stripe.com (resta in Test mode) → Developers → API keys: copia Publishable key (pk_test_…) e Secret key (sk_test_…).

2. Avvia l’inoltro dei webhook

In un terminale separato, con la Stripe CLI:

stripe login
stripe listen --forward-to http://localhost:5080/api/stripe/webhook

Il comando stampa un webhook signing secret whsec_…: copialo.

3. Inserisci le chiavi nel pannello

  1. Lascia il toggle su TEST (sandbox).
  2. Compila l’ambiente TEST: Publishable key, Secret key, Webhook secret.
  3. Premi Salva impostazioni.
Test vs LiveLe chiavi Test e Live sono due set separati: cambiare modalità non cancella l’altro ambiente. I segreti sono cifrati a riposo; i campi password vuoti non sovrascrivono i valori salvati.

Creare i piani

Menu Piani (/PlanAdmin). Creare un piano lo sincronizza con Stripe (Product + Price).

  1. Compila Nome (es. Pro), Prezzo (es. 19.99), Valuta (eur), Periodo (Mensile/Annuale).
  2. Premi Crea e sincronizza.
  3. Se riesce, accanto al piano compare un price_… (creato su Stripe).
Se manca il priceIl piano è stato salvato ma la sincronizzazione Stripe è fallita: verifica la Secret key in Stripe, poi premi Sincronizza sulla riga del piano.

Per offrire lo stesso prodotto mensile e annuale, crea due piani distinti.

Aggiungere un server

Menu Server (/ServerAdmin). I server sono le regioni tra cui i tenant scelgono dove ospitare i progetti.

  1. Nome (es. EU Milano) e Regione (es. it-mil-1).
  2. Tipo: VPS Docker (AWS ECS è uno stub non ancora implementato).
  3. Limite N: quanti progetti condividono un singolo container Postgres prima di crearne uno nuovo.
  4. Config JSON: serve solo in modalità Docker (vuoto in Logging).
  5. Spunta Abilitato e crea.

Config JSON (solo modalità Docker)

{
  "dockerHost": "tcp://IP_VPS:2376",
  "network": "traefik",
  "appImage": "registry.example.com/aski-ticketing:latest",
  "postgresImage": "postgres:16-alpine",
  "certResolver": "le",
  "entrypoint": "websecure",
  "pgAdminUser": "postgres",
  "pgAdminPassword": "***",
  "domainSuffix": "aski.app"
}
Limite N e poolRaggiunto N progetti su un container Postgres, il sistema ne crea automaticamente un altro. L’assegnazione è protetta da concorrenza (nessun superamento di N).

Tenant e progetti

Menu Tenant (/Portal).

  1. Registra azienda: ragione sociale + email di fatturazione.
  2. Apri la scheda del tenant.
  3. Nuovo progetto: nome, server/regione (tra quelli abilitati), sottodominio, eventuale dominio personalizzato.

Il progetto nasce non provisionato. Diventa Running dopo l’acquisto (webhook) o premendo Provisiona.

Acquisto & webhook

Dalla scheda tenant, sezione Acquista un piano → Checkout.

  1. Si apre Stripe Checkout: paga con la carta di test 4242 4242 4242 4242 (data futura, CVC qualsiasi).
  2. Stripe invia gli eventi al webhook (richiede stripe listen attivo).
  3. L’abbonamento passa ad Attivo e parte il provisioning del progetto collegato.
Evento StripeStato abbonamentoAzione
invoice.paid / subscription attivaAttivoProvisiona / Riavvia
subscription past_dueInsolutoSospende (dati salvi)
subscription deletedCancellatoFerma (dati conservati)

Il pulsante Gestisci fatturazione apre lo Stripe Customer Portal del tenant (cambio carta, disdetta).

Prova rapida senza Stripe

Per vedere un progetto diventare Running senza account Stripe né VPS (modalità Logging):

  1. Server: crea EU Milano / it-mil-1, tipo VPS Docker, Config JSON vuoto, N=10, Abilitato.
  2. Tenant: registra Acme Srl + email.
  3. Progetto: aprilo, crea Supporto sul server, sottodominio acme.
  4. Premi Provisiona sul progetto → stato Running.
Zero dipendenzeNessun account Stripe, nessuna VPS: utile per demo e per capire i flussi.

Problemi comuni

In dashboard vedo “Stripe non configurato”
Vai su Stripe e salva almeno le chiavi dell’ambiente attivo (Secret + Publishable + Webhook secret).
Il piano è creato ma non ha il price_…
La Secret key Stripe è errata o assente. Correggila in Stripe, poi premi Sincronizza sul piano.
Il Checkout dà errore
Il piano non è sincronizzato (manca il price) oppure le chiavi Stripe non sono valide.
Ho pagato ma il progetto non diventa Running
Verifica che stripe listen sia attivo e che esista un progetto collegato all’abbonamento. In alternativa usa il pulsante Provisiona.
In modalità Docker il provisioning fallisce
Il Control Plane non raggiunge il demone Docker o Traefik: controlla dockerHost, la rete e il certresolver nel Config JSON del server.
Come passo da sandbox a produzione?
Inserisci le chiavi LIVE nell’apposito riquadro e sposta il toggle su LIVE. Le chiavi Test restano salvate.

Sicurezza

  • I segreti Stripe sono cifrati a riposo; la firma dei webhook è verificata e gli eventi sono idempotenti.
  • Le operazioni di sospensione/stop non cancellano i dati: fermano i container.
  • ⚠️ In questa versione il Control Plane non ha ancora autenticazione: proteggilo (rete privata / reverse proxy con login) prima di esporlo. Dettagli in docs/08-sicurezza.md.
Prima del deployCambia le credenziali di default, sposta i segreti su variabili d’ambiente/secret manager, abilita HTTPS e limita l’accesso al pannello.
